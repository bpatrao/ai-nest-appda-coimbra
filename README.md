# APPDA Coimbra

Website for **APPDA Coimbra** — _Associação Portuguesa para as Perturbações do
Desenvolvimento e Autismo de Coimbra_, a non-profit supporting autistic people
and their families in the Coimbra region. Built for the **Vibe for Good**
initiative.

## Tech stack

- **[TanStack Start](https://tanstack.com/start)** + **[TanStack Router](https://tanstack.com/router)** — file-based routing with SSR / prerendering
- **React 19** + **TypeScript**
- **Vite 8** (via `@lovable.dev/vite-tanstack-config`)
- **Tailwind CSS v4** + **shadcn/ui** (Radix primitives)
- **TanStack Query**, **react-hook-form**, **zod**
- Built-in **PT/EN i18n** and an accessibility preferences panel (text size, motion, contrast, etc.)
- **Bun** as package manager / runtime

## Building a static-only site for Cloudflare Pages

This site has **no runtime server dependency**. The only server-side code is
unused boilerplate generated by the template:

- `src/lib/api/example.functions.ts` — an example `createServerFn` (`getGreeting`) that is **never called** anywhere.
- `src/lib/config.server.ts` — an unused server-config helper.
- `src/server.ts` / `src/start.ts` — SSR error-handling wrappers (only relevant when running as a server).

Forms (e.g. the service-enrolment dialog in `src/components/EnrollDialog.tsx`)
submit via `mailto:` links — purely client-side. Because nothing invokes a
server function or loader at runtime, the whole site can be **prerendered to
static HTML** and hosted on any static host.

### How the static build works

`vite.config.ts` adds a `STATIC_BUILD=1` mode that:

1. **Disables Nitro** (`nitro: false`) — so no Cloudflare Worker / server bundle is emitted.
2. **Prerenders every route** to HTML (`prerender.crawlLinks` follows in-app navigation; all 8 routes are also listed explicitly as a safety net).

> Note: outside a Lovable sandbox the `@lovable.dev` config already skips Nitro,
> so a normal `bun run build` is "Vite-only" too — but `build:static` additionally
> turns on prerendering, which is what produces the deployable HTML.

### Build it

```bash
bun run build:static
# equivalent to: STATIC_BUILD=1 vite build
```

Output lands in **`dist/client/`**:

```
dist/client/
├── index.html            # /
├── sobre/index.html      # /sobre
├── servicos/index.html   # …one folder per route
├── ...
├── assets/               # hashed JS + CSS bundles
├── images/               # site images
├── robots.txt
└── sitemap.xml
```

`dist/server/` is also produced (the build uses it internally to prerender) but
is **not** needed for deployment — only upload `dist/client/`.

Each page is real prerendered HTML (good for SEO and no-JS users); the React app
then hydrates and takes over client-side navigation.

### Verify locally

```bash
cd dist/client
python3 -m http.server 8099
# open http://localhost:8099
```

### Deploy to Cloudflare Pages

**Option A — Dashboard / Git integration**

In your Cloudflare Pages project settings:

| Setting | Value |
| --- | --- |
| Build command | `bun run build:static` |
| Build output directory | `dist/client` |
| (optional) Environment variable | `STATIC_BUILD=1` |

Setting the build command to `build:static` already injects `STATIC_BUILD=1`,
so the extra env var is only needed if you call `vite build` directly.

**Option B — Direct upload with Wrangler**

```bash
bun run build:static
bunx wrangler pages deploy dist/client --project-name appda-coimbra
```

No `_worker.js`, no Functions, no `_redirects` are required — every route is a
real file, and Cloudflare Pages serves `/sobre` from `/sobre/index.html`
automatically.

### Assets are self-contained

The build has **no external asset dependencies**. The logo lives at
`public/appda-coimbra-logo.png` and is referenced as `/appda-coimbra-logo.png`
in `src/components/site-header.tsx`; all other images are in `public/images/`.
Everything is copied into `dist/client/` at build time, so nothing is loaded
from Lovable's CDN at runtime.

> The original template referenced the logo via Lovable's asset system
> (`src/assets/appda-coimbra-logo.png.asset.json` → `/__l5e/...`), which only
> resolves on Lovable's infrastructure. That descriptor is no longer imported;
> if you swap the logo, replace `public/appda-coimbra-logo.png`.

## Project layout

```
src/
├── routes/        # file-based routes (one file per page) + __root.tsx shell
├── components/    # site header/footer, dialogs, mode/lang toggles
│   └── ui/        # shadcn/ui primitives
├── i18n/          # PT/EN dictionary + useT() hook

├── server.ts      # SSR error wrapper (server only)

```
